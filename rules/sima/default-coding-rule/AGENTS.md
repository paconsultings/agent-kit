# AGENTS.md

## Repo Defaults
- Use `rg` for text/file search over slower alternatives.
- Keep changes minimal and scoped to the requested task.
- Prefer non-destructive operations; avoid forceful git commands.
- Run targeted tests for touched modules before finalizing.

## Coding Standards
- Preserve existing project structure and naming conventions.
- Add concise comments only where logic is non-obvious.
- Prefer explicit error handling and actionable error messages.

## PR / Review Expectations
- Include a short risk summary for behavior changes.
- Mention tests run and any untested paths.
- Call out follow-up work if implementation is intentionally partial.
